# タンク + 水中ポンプ セット導入マニュアル

概要

このページは、本ソフトウェア（ina-device-hub）を水槽／タンクと水中ポンプを組み込んで運用する際の実務マニュアルです。ハードウェアの選定、取り付け、電気・配管の注意点、運用と保守、ソフトウェアとの連携ガイド（推奨パターン）を含みます。

購入リスト（参考）

以下の Amazon ウィッシュリストにある商品の組合せを前提としています（必須ではありませんが、入手・互換性確認が容易です）。

- Amazon ウィッシュリスト: https://www.amazon.co.jp/hz/wishlist/ls/84LGOV9KCZVD?ref_=list_d_wl_lfu_nav_2

（ウィッシュリスト内の「タンク」「水中ポンプ」「ホース」「電源タップ」等を組み合わせて使用してください）

想定ユースケース

- 小〜中規模の循環/給水/排水用途（アクアリウム、養殖ミニタンク、ラボ用循環系など）。
- 定期撮影やセンサーデータと連動してポンプを制御する（例：一定条件でポンピングを開始/停止）。

前提と注意（重要）

- 水中ポンプは一般に水中で動作することを前提とした製品です。必ず取扱説明書を読み、定格（電圧、電流、流量、揚程）を確認してください。
- 電源は家庭用交流（AC）製品が多く、感電や漏電の危険があります。屋外や湿気のある場所では必ず漏電遮断器（RCD/GFCI）を経由してください。
- 電気工事、据付が不明な場合は必ず有資格者（電気工事士）に依頼してください。

機材の例（ウィッシュリストから取る順序）

1. タンク（容量は用途に合わせる）
2. 水中ポンプ（定格電圧・最大流量・揚程を合わせて選定）
3. ホース（内径がポンプ吐出口に合うもの）
4. フィルタやストレーナ（ポンプ吸入口のごみ混入防止）
5. 電源タップ（防滴仕様または屋内専用。RCD 推奨）
6. チューブバンド、クランプ、フッ素テープ（配管のシール用）

設置手順（機械・配管）

1. 設置場所を決める
   - タンクが水平で安定した台に置かれていること、周囲に水が漏れても問題ないか確認。
   - ポンプの吸入口が底泥やゴミを吸わないようにする（ストレーナを付ける）。
2. ポンプの固定
   - タンク内に沈める際はポンプ底面が安定するように位置決め。
   - 振動が伝わらないよう、ゴムマウントなどを検討。
3. 配管接続
   - ホースは無理に曲げず、急曲を避ける。接続部はクランプで確実に固定。
   - 空気噛み対策：吸い込み側に高い位置の空気ポケットがないことを確認。
4. 電源配線
   - 防水コンセントや屋内コンセントから供給。屋外設置は防水ボックスと漏電遮断を必須とする。
   - スイッチやタイマー制御を入れる場合は、定格に合った中継リレー/ソリッドステートリレー（SSR）を用いる。

試運転（初期動作確認）

- 組み上げ後、通水前にホース結合やクランプの緩みがないか確認。
- 電源投入前に周囲に水のかかるものが無いか確認。
- 電源を入れ、空回りや異音が無いか数秒観察。問題なければ水を満たして再度動作確認。
- 流量・揚程が想定通りか確認し、必要に応じて配管長の見直しやポンプ交換を検討。

保守・メンテナンス

- 定期点検（週次〜月次）:
  - 吸入口のストレーナとホースの詰まりチェック。
  - 電源ケーブルの被覆・プラグの状態確認。
  - 異音や振動の有無、発熱の有無の確認。
- 年次点検:
  - ポンプ本体の分解清掃（説明書に従う）。
  - 必要に応じて交換パーツ（インペラなど）を事前に用意。

ソフトウェアとの連携（運用者向け提案）

注意: 現行の ina-device-hub コードベースはセンサーデータ受信と画像保存を主目的として実装されています。アクチュエータ（ポンプ）制御用の API は明示的に含まれていないため、以下は運用上の推奨パターンです（実装の追加が必要になります）。

1. ハードウェア制御アーキテクチャ（推奨）
   - ポンプの ON/OFF は低電圧制御（例: 3.3V/5V の GPIO 経由でリレーを駆動）または中間にリレー制御ユニット（RPI/ESP32+リレーモジュール）を置く。
   - 直接高電圧をサーバに接続しないこと（危険）。必ず絶縁リレーや SSR を使う。
2. 通信パターン（MQTT 推奨）
   - HubMQTTClient を経由する形で以下のトピック設計を推奨します（例）:
     - 制御送信: `actuator/<device_id>/pump/set`  payload: `{"state":"ON"}` または `{"state":"OFF"}`
     - 状態報告: `actuator/<device_id>/pump/state` payload: `{"state":"ON","updated_at":"..."}`
   - ハードウェア側（Raspberry Pi, ESP32 等）が該当トピックを subscribe してリレーを操作し、操作結果を状態トピックで publish する。
3. ina-device-hub 側の統合案
   - 既存の `HubMQTTClient` と `SensorDataQueue` を利用して、ポンプ制御トピックを受け渡しするコネクタを追加する（例: `actuator_queue` を作り、コマンドを queue に入れて別スレッドで実行）。
   - Web 管理画面に「ポンプ手動制御」ボタンを追加して、バックエンドから MQTT publish する仕組みを作ると運用が容易になる。

トラブルシュート（よくある事象）

- ポンプが動かない
  - 電源供給とプラグを確認
  - 吸入口が詰まっていないか確認
  - リレーや制御回路の断線を確認
- 異音・振動
  - 異物混入、インペラの破損を疑う。速やかに停止して点検。
- 流量不足
  - 配管の曲がり、目詰まり、吐出口の閉塞、またはポンプの定格不足を確認

補足（安全とコンプライアンス）

- 動力配線を触る際は必ず電源を切り、二度と通電しないことを確認して作業する。
- 法令・施設ルールに従い、必要があれば防爆仕様や産業用グレード機器を選定する。
